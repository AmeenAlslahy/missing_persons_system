# توثيق API - نظام البحث عن الأشخاص المفقودين

## نظرة عامة

هذا النظام يوفر API شامل لإدارة البحث عن الأشخاص المفقودين باستخدام الذكاء الاصطناعي. النظام يدعم أربعة أنواع من المستخدمين مع صلاحيات مختلفة.

## أنواع المستخدمين وصلاحياتهم

### 1. المستخدم العادي (user)
- **الوصف**: مستخدم عادي يمكنه البحث عن المفقودين وإنشاء بلاغات
- **الصلاحيات**:
  - إنشاء بلاغات مفقود/معثور
  - البحث في البلاغات
  - تحديث ملفه الشخصي
  - تلقي الإشعارات

### 2. المتطوع (volunteer)
- **الوصف**: متطوع يساعد في عمليات البحث والتنسيق
- **الصلاحيات**: جميع صلاحيات المستخدم العادي +
  - مراجعة البلاغات
  - المشاركة في عمليات المطابقة
  - الوصول لإحصائيات محدودة
  - إدارة المناطق الجغرافية

### 3. المشرف (admin)
- **الوصف**: مشرف يدير النظام والمستخدمين
- **الصلاحيات**: جميع صلاحيات المتطوع +
  - إدارة المستخدمين (توثيق، حظر، تغيير الأدوار)
  - مراجعة وإدارة جميع البلاغات
  - إدارة إعدادات المطابقة
  - الوصول للتحليلات والإحصائيات الكاملة
  - إدارة الإشعارات
  - مراجعة سجلات التدقيق

### 4. المشرف الرئيسي (super_admin)
- **الوصف**: مشرف رئيسي له صلاحيات كاملة
- **الصلاحيات**: جميع صلاحيات المشرف +
  - إدارة المشرفين الآخرين
  - الوصول لجميع البيانات والإعدادات
  - إدارة النظام بالكامل

## المصادقة والأمان

### طرق المصادقة
- **JWT (JSON Web Tokens)**: للمصادقة في API
- **Session Authentication**: للواجهات الإدارية
- **Token Refresh**: لتجديد التوكنات

### رؤوس الطلبات المطلوبة
```
Authorization: Bearer <access_token>
Content-Type: application/json
```

## API Endpoints

## 1. تطبيق الحسابات (Accounts)

### المصادقة

#### تسجيل مستخدم جديد
```
POST /api/accounts/register/
```
**الصلاحيات**: الجميع (غير مصادق عليهم)
**المدخلات**:
```json
{
  "email": "user@example.com",
  "full_name": "اسم المستخدم الكامل",
  "phone": "رقم الهاتف",
  "password": "كلمة المرور",
  "date_of_birth": "1990-01-01",
  "gender": "M",
  "governorate": "المحافظة",
  "district": "المديرية"
}
```
**المخرجات**:
```json
{
  "user": {...},
  "refresh": "refresh_token",
  "access": "access_token",
  "message": "تم إنشاء الحساب بنجاح"
}
```

#### تسجيل الدخول
```
POST /api/accounts/login/
```
**الصلاحيات**: الجميع (غير مصادق عليهم)
**المدخلات**:
```json
{
  "email": "user@example.com",
  "password": "كلمة المرور"
}
```
**المخرجات**: نفس مخرجات التسجيل

#### تسجيل الخروج
```
POST /api/accounts/logout/
```
**الصلاحيات**: المصادق عليهم
**المدخلات**:
```json
{
  "refresh": "refresh_token"
}
```

#### تجديد التوكن
```
POST /api/accounts/token/refresh/
```
**الصلاحيات**: المصادق عليهم
**المدخلات**:
```json
{
  "refresh": "refresh_token"
}
```

### إدارة الملف الشخصي

#### عرض الملف الشخصي
```
GET /api/accounts/profile/
```
**الصلاحيات**: المصادق عليهم

#### تحديث الملف الشخصي


**المدخلات**: بيانات المستخدم المحدثة

### التحقق من الهوية

#### طلب التحقق
```

```
**الصلاحيات**: المصادق عليهم
**المدخلات**:
```json
{
  "national_id": "رقم الهوية الوطنية"

```

### إدارة المتطوعين

#### قائمة المتطوعين
```
GET /api/accounts/volunteers/
```
**الصلاحيات**: المصادق عليهم

#### تفاصيل متطوع
```
GET /api/accounts/volunteers/{id}/
```
**الصلاحيات**: المصادق عليهم

### إدارة المستخدمين (للمشرفين)

#### قائمة المستخدمين
```
GET /api/accounts/users/
```
**الصلاحيات**: المشرفون فقط
**المعاملات**:
- `verification_status`: حالة التحقق
- `user_role`: دور المستخدم
- `search`: البحث في الاسم والبريد

#### تحقق من مستخدم
```
POST /api/accounts/admin/verify/{user_id}/
```
**الصلاحيات**: المشرفون فقط
**المدخلات**:
```json
{
  "action": "accept|reject",
  "notes": "ملاحظات المشرف"
}
```

#### إجراءات المشرف على المستخدمين
```
POST /api/accounts/users/{id}/admin_action/
```
**الصلاحيات**: المشرفون فقط
**المدخلات**:
```json
{
  "action": "block|unblock|verify|reject_verification|change_role|recalculate_trust",
  "reason": "سبب الإجراء",
  "new_role": "الدور الجديد (اختياري)"
}
```

## رموز الحالة الشائعة

### رموز النجاح
- `200 OK`: تم تنفيذ الطلب بنجاح
- `201 Created`: تم إنشاء المورد بنجاح
- `204 No Content`: تم الحذف بنجاح

### رموز الأخطاء
- `400 Bad Request`: بيانات الطلب غير صحيحة
- `401 Unauthorized`: غير مصادق عليه
- `403 Forbidden`: لا توجد صلاحية
- `404 Not Found`: المورد غير موجود
- `500 Internal Server Error`: خطأ في الخادم

## أمثلة على الاستخدام

### إنشاء بلاغ مفقود
```bash
curl -X POST http://localhost:8000/api/reports/reports/ \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "report_type": "missing",
    "person_name": "أحمد محمد علي",
    "age": 25,
    "gender": "M",
    "last_seen_location": "شارع الملك فيصل، صنعاء",
    "last_seen_date": "2024-01-15",
    "contact_phone": "+967-1-234567",
    "contact_person": "محمد أحمد علي"
  }'
```

### البحث في البلاغات
```bash
curl -X GET "http://localhost:8000/api/reports/reports/?search=أحمد&report_type=missing" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

### مراجعة مطابقة
```bash
curl -X POST http://localhost:8000/api/matching/matches/123/review/ \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "decision": "accept",
    "notes": "تم التحقق من الهوية والتأكد من التطابق"
  }'
```
